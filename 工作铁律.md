# 工作铁律 - 必须严格遵守

*版本：v1.0 | 生效时间：2026-03-05*

---

## ⚠️ 优先级最高 - 违反可能导致严重后果

### 1️⃣ openclaw.json 修改三步铁律

```
改前备份（带时间戳）
    ↓
改前查文档确认字段合法值
    ↓
改后做双验证（JSON 解析 + openclaw doctor）通过后才能重启
```

**违反后果**：配置损坏、服务无法启动

---

### 2️⃣ 禁止危险重启动作

❌ **禁止** 先 kill 前台 gateway 再 systemd start
❌ **禁止** stop+start 快速连击
✅ **优先** restart，并在校验通过后执行

**正确流程**：
```bash
# 1. 校验配置
openclaw doctor

# 2. 重启服务
openclaw gateway restart

# 3. 验证服务
openclaw gateway status
```

---

### 3️⃣ 禁止猜命令/猜配置

❌ **禁止** 不熟悉命令直接执行
❌ **禁止** 配置字段靠猜

**正确做法**：
```bash
# 查帮助
command --help

# 查文档
openclaw help
# 或查阅 /home/ubuntu/.npm-global/lib/node_modules/openclaw/docs
```

---

### 4️⃣ 给选项后必须等确认

❌ **禁止** 擅自拍板执行未确认的方案

**正确流程**：
```
1. 分析需求
2. 给出 2-3 个可行方案
3. 说明各方案优缺点
4. 等待用户明确确认
5. 执行确认的方案
```

---

### 5️⃣ 密钥安全铁律

❌ **禁止** 在输出里暴露任何密钥
✅ **必须** 通过 1Password op 读取
✅ **示例** 里只用占位符，不写真值

**错误示例**：
```bash
# ❌ 错误：暴露密钥
SMTP_PASS=wusulfxbhqyqcbce
```

**正确示例**：
```bash
# ✅ 正确：占位符
SMTP_PASS=<your_auth_code>

# ✅ 正确：从 1P 读取
op read "vault/item/field"
```

---

### 6️⃣ 1Password SSH 调用铁律（强制）

✅ **必须** 在 tmux 里跑 op 相关操作
✅ **私钥** 只进 ssh-agent，不落盘
✅ **连接服务器** 统一走 1P op 取密钥

**正确流程**：
```bash
# 1. 进入 tmux
tmux new -s ssh-session

# 2. 从 1P 获取密钥并添加到 agent
op read "vault/ssh/item" | ssh-add -

# 3. 连接服务器
ssh user@host
```

### 7️⃣ 代码/生产变更流程

```
本地修改
    ↓
测试验证
    ↓
git commit
    ↓
等待用户确认
    ↓
推送/部署
```

❌ **禁止** 直接在在线服务器改核心代码

## 📋 检查清单

执行重要操作前，逐项确认：

- [ ] 是否需要备份？备份了吗？
- [ ] 是否查过文档/帮助？
- [ ] 配置字段是否确认过合法值？
- [ ] 是否给了用户选项并等待确认？
- [ ] 是否涉及密钥？是否安全处理？
- [ ] 是否需要用 1Password？是否在 tmux 中执行？
- [ ] 代码变更是否走了完整流程？
